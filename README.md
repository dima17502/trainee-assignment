# Автотесты для микросервиса объявлений Avito  

Данный репозиторий содержит:
- оформленные вручную тест-кейсы (`TESTCASES.docx`)
- комплект автотестов на `pytest` для проверки API
- инструкцию по запуску и установке окружения
- примеры фактического поведения API

---
## Ограничение по автотестам

Автоматические тесты покрывают позитивные (“happy path”) сценарии, которые
наиболее стабильны и воспроизводимы на предоставленном учебном стенде.

Негативные сценарии (невалидные UUID, некорректные sellerId, ошибки типов,
пустые тела запросов, неверные форматы JSON и т.п.) намеренно не были 
автоматизированы по следующим причинам:

1. Фактическое поведение API в ряде негативных сценариев расходится
   с документацией и ожидаемым результатом.
2. На учебном стенде часть ошибок обрабатывается нестабильно
   (например, разные форматы ответа, разное тело ответа, отсутствие единообразного JSON).
3. Для автотестов важна предсказуемость результатов, чтобы весь комплект тестов 
   всегда проходил зелёным и отражал реальные гарантированные возможности API.

Все негативные сценарии при этом подробно проработаны и задокументированы 
в файле [TESTCASES.md](/Task%202/TESTCASES.docx).


## Описание проекта

API позволяет:
- создавать объявления
- получать объявление по id
- получать список объявлений по sellerId
- получать статистику объявления

В рамках задания требовалось:
1. Проанализировать документацию и написанные ручные тест-кейсы  
2. Покрыть сценарии автотестами  
3. Оформить результат в виде репозитория  

---


- `requirements.txt` — библиотеки для автотестов  
- `conftest.py` — фикстуры (session, base_url, seller_id, created_item)  
- `test_items_api.py` — сами автотесты  
- `TESTCASES.md` — ручные тест-кейсы 

## Запуск автотестов в IDE
- Скачать необходимые библиотеки
```
pip install -r requirements.txt
```
- Запустить автотесты из корневого репозитория
```
pytest -v
```
- Получить HTML страницу с результатами тестирования
```
pytest --html=report.html --self-contained-html
```
